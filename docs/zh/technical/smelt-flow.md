│  mlx-zig chat --model ~/models/deepseek-v4-flash-4bit --prompt "Hello"
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 解析命令行参数                                            │
│    smelt = false                                            │

└─────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 创建 SmeltConfig                                         │
│    SmeltConfig {                                            │

└─────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────┐

│    if (smelt.enabled and smelt.load_fraction < 1.0) {      │

│    // ❌ 条件不满足，smelt_mask = null                      │

## 使用 --smelt 的成功流程

```
用户命令
│
│  mlx-zig chat --model ~/models/deepseek-v4-flash-4bit --smelt --prompt "Hello"
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 解析命令行参数                                            │
│    smelt = true  ✅                                         │
│    smelt_experts = 1.0                                      │
└─────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 创建 SmeltConfig                                         │
│    SmeltConfig {                                            │
│        enabled: true,  ✅                                   │
│        load_fraction: 1.0                                   │
│    }                                                        │
└─────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 加载权重文件                                              │
│                                                             │
│    // 虽然 load_fraction = 1.0，但 enabled = true          │
│    // 代码会自动适应文件中实际存在的专家                     │
│                                                             │
│    for each weight in model_files {                        │
│        try {                                                │
│            weights.put(name, load(name));  ✅              │
│        } catch {                                            │
│            // 如果权重不存在，跳过（不报错）                  │
│            continue;                                        │
│        }                                                    │
│    }                                                        │
│                                                             │
│    结果：weights 包含文件中实际存在的权重                    │
│    ├─ Expert 0   ✅                                         │
│    ├─ Expert 17  ✅                                         │
│    ├─ Expert 34  ✅                                         │
│    └─ ... (38 个专家)                                       │
└─────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 构建模型 (buildDSV4Model)                                │
│                                                             │
│    n_routed_experts = 256                                  │
│                                                             │
│    // ✅ 创建 smelt_mask                                    │
│    smelt_mask = buildMask(256)                             │
│    // 根据实际加载的专家自动调整 mask                         │
│                                                             │
│    // 检查是否有专家权重                                     │
│    if (weights.get("...experts.0.w1.weight") == null) {   │
│        // 没有任何专家 → 创建 dummy SwitchGLU              │
│        return dummy_switch_glu;  ✅                        │
│    }                                                        │
│                                                             │
│    // 或者，只加载存在的专家                                 │
│    for (e = 0; e < 256; e++) {                             │
│        if (expert_exists_in_weights(e)) {                  │
│            gate_list[e] = weights.get(name);  ✅           │
│        } else {                                             │
│            // 未加载的专家，路由器会避免选择                  │
│            smelt_mask[e] = false;                          │
│        }                                                    │
│    }                                                        │
│                                                             │
│    // 将 smelt_mask 传递给 Gate                             │
│    gate.smelt_mask = smelt_mask;  ✅                       │
└─────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 推理时路由器调整                                          │
│                                                             │
│    // 计算路由分数                                            │
│    router_logits = gate.weight @ hidden_states             │
│                                                             │
│    // ✅ 对未加载的专家施加惩罚                              │
│    for (e = 0; e < 256; e++) {                             │
│        if (smelt_mask != null and !smelt_mask[e]) {       │
│            router_logits[e] -= 1000.0;  // 大幅降低概率    │
│        }                                                    │
│    }                                                        │
│                                                             │
│    // Top-K 选择                                            │
│    selected_experts = topk(router_logits, k=2)             │
│    // 结果：只会选择已加载的专家 (0, 17, 34, ...)           │
└─────────────────────────────────────────────────────────────┘
│
▼
✅ 成功：生成 token
```

## 关键差异对比

| 步骤 | 不使用 --smelt | 使用 --smelt |
|------|---------------|-------------|
| **参数解析** | `smelt = false` | `smelt = true` ✅ |
| **SmeltConfig** | `enabled = false` | `enabled = true` ✅ |
| **权重加载** | 尝试加载所有 256 个专家 | 只加载文件中存在的专家 ✅ |
| **smelt_mask** | `null` | `[true, false, false, ..., true]` ✅ |
| **模型构建** | 期望所有 256 个专家都存在 | 适应实际加载的专家数量 ✅ |
| **访问不存在的专家** | ❌ 返回 `MissingWeight` | ✅ 跳过或使用 dummy |
| **路由器** | 可能选择任何专家 | 避免选择未加载的专家 ✅ |
| **结果** | ❌ 失败 | ✅ 成功 |

## 总结

**核心问题**：4-bit 模型文件只包含部分专家（15-30%），但代码默认期望所有 256 个专家都存在。

**解决方案**：使用 `--smelt` 标志启用部分专家模式，让代码适应实际可用的专家数量。

**记忆口诀**：
- ❌ 不用 smelt → 期望全部 → 找不到 → MissingWeight
- ✅ 使用 smelt → 适应部分 → 只用存在的 → 成功运行
